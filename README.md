# MaColoc

Gamified household management app for apartment-sharing. Track chores, manage deep cleans, and keep your roommates accountable — with points, badges, and a leaderboard.

## Tech Stack

- **Frontend:** Flutter 3.x (iOS, Android, Web)
- **State Management:** Riverpod
- **Routing:** GoRouter with auth-aware redirect
- **Backend:** Firebase (Auth, Firestore, Cloud Functions v2, Storage)
- **Data Models:** freezed + json_serializable
- **Cloud Functions:** TypeScript

## Getting Started

### Prerequisites

- Flutter 3.41+ (`flutter --version`)
- Node.js 20+ (`node --version`)
- Firebase CLI (`npm install -g firebase-tools`)
- Java 21+ (for Firebase emulators)

### 1. Clone and install

```bash
cd macoloc
flutter pub get
cd functions && npm install && cd ..
```

### 2. Configure Firebase

```bash
firebase login
firebase projects:create macoloc-app --display-name "MaColoc"
flutterfire configure --project=macoloc-app --platforms=android,ios,web
```

This generates `lib/firebase_options.dart` with real credentials. Then enable **Email/Password** auth in the [Firebase Console](https://console.firebase.google.com).

### 3. Run with emulators (recommended for development)

**Terminal 1** — start emulators:
```bash
firebase emulators:start
```

**Terminal 2** — run the app:
```bash
flutter run -d chrome
```

In debug mode, the app automatically connects to local emulators (Auth :9099, Firestore :8080, Functions :5001). The emulator UI is at http://localhost:4000.

### 4. Run without emulators (production Firebase)

Deploy Cloud Functions first:
```bash
cd functions && npm run deploy
```

Then run in release/profile mode:
```bash
flutter run -d chrome --release
```

## Development

### Run tests

```bash
flutter test                    # 26 unit tests
flutter analyze                 # Static analysis
cd functions && npm run build   # TypeScript check
```

### Code generation (after modifying freezed models)

```bash
dart run build_runner build --delete-conflicting-outputs
```

> **Note:** `riverpod_generator` and `riverpod_lint` must be temporarily commented out in `pubspec.yaml` before running `build_runner` due to an upstream `analyzer_plugin` incompatibility with Dart 3.11.1. Restore them after generation.

### Adding a new model

1. Create `lib/src/models/<name>.dart` with `@freezed` annotation
2. Create `test/models/<name>_test.dart` with serialization tests
3. Run code generation
4. Run tests to verify

## Architecture

### Auth Flow

```
App Start -> Firebase Init -> authStateProvider (stream)
  |-- null (not logged in) -> /sign-in
  |-- User (no house)      -> /onboarding -> create or join
  +-- User (has house)     -> /home
```

House membership is queried from Firestore on startup (`currentHouseIdProvider`), so it survives app restarts.

### House Membership

All membership mutations go through **Cloud Functions** (callable):
- `createHouse` — transaction: house doc + admin member subdoc
- `joinHouse` — finds house by invite code, adds member
- `leaveHouse` — admin guard, recursive delete for last member
- `removeMember` — admin-only removal

Single-house enforcement: callables reject if user already belongs to a house.

### Security Rules

- All reads gated on `members[]` array membership
- Issues: field-level update guards (claim, resolve, dispute, react)
- Reactions: restricted to caller's own key
- Deep clean updates: callable-only (Sprint 3)
- Members: presence self-update only, CRUD via callables
